# RGR ML Dashboard 

**RGR ML Dashboard** – это Streamlit-приложение для инференса обученных моделей и визуализации данных на основе набора данных по отчислению студентов из UCI для бинарной задачи классификации со следующими классами: отчислен(класс 0) и завершил обучение(класс 1). Проект включает скрипт обучения моделей и интерактивный дашборд, позволяющий анализировать данные, смотреть метрики моделей и делать предсказания (по CSV или вручную). Ниже приведено подробное описание структуры проекта, инструкций по установке и использованию, а также пояснений к компонентам и рекомендаций по улучшению.

## Цели и назначение проекта  

Этот проект демонстрирует полный цикл разработки ML-приложения: от предобработки и обучения нескольких моделей (логистическая регрессия, случайный лес, XGBoost, CatBoost и др.) до визуализации результатов и предоставления интерфейса для инференса. Он использует набор данных по отчислению студентов из UCI для бинарной задачи классификации, содержащий 3630 строк и 37 признаков. Цель приложения – предсказать, является ли опухоль злокачественной или доброкачественной, а также дать пользователю возможность загружать новые данные для предсказаний. Информация о датасете и модели отображается на отдельных вкладках Streamlit, что упрощает ознакомление с результатами.

## Структура файлов и папок

Ниже приведена таблица с описанием основных файлов и каталогов репозитория:

| Файл/Папка              | Описание                                                                                            |
|-------------------------|-----------------------------------------------------------------------------------------------------|
| `train_models.py`       | Скрипт обучения ML-моделей: загрузка данных, предобработка, обучение (LogReg, RF, XGBoost, CatBoost и др.), вычисление метрик и сохранение лучших результатов в `artifacts/`. |
| `app.py`                | Streamlit-приложение с четырьмя вкладками: «Разработчик», «О датасете», «Визуализации», «Предсказание». Отвечает за отображение интерфейса. |
| `artifacts/`            | Артефакты обучения: файлы с метриками (например, JSON/CSV с результатами), сохранённый препроцессор (scikit-learn Pipeline или т.п.), объект лучшей модели. |
| `models/`               | Сериализованные модели машинного обучения. Модели XGBoost и CatBoost сохраняются их встроенными методами в формате `.json` или `.cbm`【46†L128-L136】, а модели scikit-learn (LogReg, RandomForest и др.) – через `pickle` (например, `best_model.pkl`). |
| `data/data_clf.csv` | Пример входных данных в формате CSV (соответствует исходным признакам набора). Предназначен для демонстрации загрузки новых данных в приложении. |

Каждый компонент играет роль в общем пайплайне: `train_models.py` подготавливает и сохраняет модели, а `app.py` подгружает эти артефакты для интерактивной работы в веб-интерфейсе.

## Установка и запуск

Для подготовки окружения рекомендуется использовать виртуальное окружение. Например: 

```bash
python -m venv venv           # создание виртуального окружения (в каталоге venv)
source venv/bin/activate      # активация окружения (Linux/Mac)
venv\Scripts\activate.bat     # активация (Windows)
``` 

После активации установите зависимости:

```bash
pip install --upgrade pip
pip install -r requirements.txt
```  

Это установит все необходимые библиотеки (Streamlit, pandas, scikit-learn, xgboost, catboost и др.) согласно `requirements.txt`. Затем запустите обучение моделей командой:

```bash
python train_models.py
```

Этот скрипт произведёт загрузку данных, обучение нескольких моделей, сравнит их по метрикам (например, Accuracy, Precision, Recall, F1-score) и сохранит результаты в каталог `artifacts/` и `models/`. После успешного обучения стартуйте Streamlit-приложение:

```bash
streamlit run app.py
```

Откроется локальный веб-сервер, обычно доступный по адресу `http://localhost:8501`. Убедитесь, что в `train_models.py` и `app.py` указаны правильные пути к артефактам моделей (обычно относительные пути, если запуск из корня проекта).

## Интерфейс приложения Streamlit

Streamlit-приложение содержит четыре вкладки, доступные через боковое меню:

- **Разработчик (Developer):** В этой вкладке отображается информация об авторе проекта (ФИО, группа, фото и т.д.). Здесь можно заменить плейсхолдеры на свои данные. Обычно это статичный текст/изображение, поясняющее, кто разработал приложение.

- **Визуализации (Visualizations):** Графический анализ данных. Часто здесь строятся гистограммы, боксплоты, correlation matrix и другие диаграммы для признаков и целевой переменной. Это помогает понять структуру данных (например, распределение признаков для меток M/B). Streamlit позволит интерактивно выбирать графики.

- **Предсказание (Prediction):** Основная функциональность: пользователь может загрузить CSV-файл с новыми данными или вручную ввести значения признаков через форму, чтобы получить прогноз модели. Здесь отображаются результаты (вероятность и метка «M/B»). Форма проверки CSV обычно реализована через `st.file_uploader(accept=".csv")`, что позволяет загружать файлы только в формате CSV【55†L1-L4】. Также рекомендуется сделать валидацию: например, проверять наличие всех ожидаемых столбцов и их типы. В справке к вкладке укажите формат CSV (см. следующий раздел).

### Формат CSV для предсказания

| Колонка                | Тип   | Описание                                                 |
|------------------------|-------|----------------------------------------------------------|
| `id`                   | int64 | Уникальный идентификатор образца (при наличии).         |
| `radius_mean`          | float | Средний радиус ядра (усреднённое расстояние от центра). |
| `texture_mean`         | float | Среднее значение текстуры (стандартное отклонение).    |
| `perimeter_mean`       | float | Средний периметр ядра.                                  |
| `area_mean`            | float | Средняя площадь ядра.                                   |
| `smoothness_mean`      | float | Среднее значение гладкости ядра.                       |
| ... и т.д. (всего 30 признаков) | ...   | Расширение аналогичных признаков (compactness, concavity и др.) |
| `diagnosis`            | —     | **Не включается** при инференсе (целевой признак).      |

*Примечание:* в обучающих скриптах `diagnosis` (целевая метка) использовался как метка для обучения, а при реальном использовании модели его не должно быть в поданных данных. При загрузке CSV код проверяет, что все необходимые признаки присутствуют. Streamlit можно настроить так, чтобы **при загрузке невалидного CSV** появлялось уведомление об ошибке.  

## Тренировочный пайплайн и выбор лучшей модели

На этапе обучения (`train_models.py`) реализован классический ML-пайплайн:

```mermaid
flowchart TD
    RawData[Raw Dataset]
    RawData --> Clean[Preprocessing]
    Clean --> Split[Train/Test Split]
    Split --> TrainModels[Train Models (LogReg, RF, XGB, CatB,...)]
    TrainModels --> Eval[Compute Metrics (Accuracy, Recall, F1, AUC...)]
    Eval --> Select[Select Best Model]
    Select --> Save[Save Best Model + Preprocessor]
    Save --> Artifacts[Artifacts (metrics.json, *.pkl, *.json)]
```

- **Предобработка (Preprocessing):** включает масштабирование (StandardScaler) и/или кодирование категориальных признаков (если были). Можно использовать `sklearn.pipeline.Pipeline` и `ColumnTransformer` для упрощения. Препроцессор обучается на тренировочных данных и сохраняется в `artifacts/` для последующего применения к новым данным.

- **Обучение моделей (Training):** несколько моделей обучаются на тренировочном сете. Для каждого сохраняются показатели (accuracy, precision, recall, F1-score и ROC AUC) на валидации/тесте.  

- **Выбор лучшей модели:** осуществляется по заранее выбранному критерию (например, максимальный ROC AUC или F1 на валидации). Результаты сравнения обычно сохраняются в таблице (CSV/JSON) в `artifacts/`. Лучшая модель сериализуется. 

**Метрики:** частые метрики для этой задачи – точность (accuracy), полнота (recall), точность (precision), F1-score и ROC AUC. Пример таблицы с метриками для каждой модели:

| Модель        | Accuracy | Precision | Recall | F1-score | ROC AUC |
|---------------|----------|-----------|--------|----------|---------|
| LogReg        | 0.XX     | 0.XX      | 0.XX   | 0.XX     | 0.XX    |
| RandomForest  | 0.XX     | 0.XX      | 0.XX   | 0.XX     | 0.XX    |
| XGBoost       | 0.XX     | 0.XX      | 0.XX   | 0.XX     | 0.XX    |
| CatBoost      | 0.XX     | 0.XX      | 0.XX   | 0.XX     | 0.XX    |

(Цифры заменяются реальными результатами после запуска `train_models.py`.) Лучшая модель маркируется (например, звездочкой) и сохраняется в `models/`.

**Сохранение моделей:** XGBoost и CatBoost используют встроенные методы `.save_model()`, например:
```python
xgb_model.save_model('models/best_xgb.json')
catboost_model.save_model('models/best_catboost.cbm')
```
Эти форматы сохраняют только параметры модели【46†L128-L136】. Модели sklearn (LogReg, RandomForest и др.) сохраняются с помощью `pickle`:
```python
import pickle
pickle.dump(best_model, open('models/best_model.pkl','wb'))
```
Важно отметить, что **pickle-формат не защищён и не гарантирует переносимость** между разными версиями Python/скриптов【46†L199-L204】. Поэтому для долгосрочного хранения моделей лучше использовать нативные форматы библиотек (JSON для XGBoost, CBM для CatBoost)【46†L128-L136】 и сохранять код препроцессора отдельно.

## Повторимость результатов

Для воспроизводимости результатов следует фиксировать *случайные зерна* (random seed) во всех библиотеках (numpy, scikit-learn, XGBoost, CatBoost и др.). Это позволяет при повторном запуске скрипта получать идентичные метрики【51†L119-L122】. Например:

```python
import numpy as np
import random
from sklearn.utils import shuffle

SEED = 42
np.random.seed(SEED)
random.seed(SEED)
# sklearn: многие функции имеют параметр random_state=SEED
# catboost: в CatBoostClassifier(random_seed=SEED)
```
## Пример использования артефактов и предсказания

После обучения можно загружать сохранённые объекты в коде. Пример на Python для инференса:

```python
import pickle
import pandas as pd

# Загрузка препроцессора и модели
preprocessor = pickle.load(open('artifacts/preprocessor.pkl','rb'))
model = pickle.load(open('models/best_model.pkl','rb'))

# Чтение нового набора данных (csv без колонки diagnosis)
df_new = pd.read_csv('data/breast_cancer_sample.csv')

# Применение препроцессинга
X_new = preprocessor.transform(df_new)

# Предсказание вероятностей и меток
y_proba = model.predict_proba(X_new)[:,1]  # вероятность класса "M"
y_pred = model.predict(X_new)
```

В Streamlit-интерфейсе эта логика инкапсулирована: при загрузке CSV или заполнении полей формы выполняются аналогичные действия, а результаты отображаются в таблице/графике. Для загрузки файла используется `st.file_uploader("Upload CSV", type="csv")` (ограничение по типу обеспечивает только файлы CSV)【55†L1-L4】. 

**Пример кода** загрузки CSV и проверки столбцов:

```python
uploaded_file = st.file_uploader("Upload CSV", type="csv")
if uploaded_file is not None:
    df_input = pd.read_csv(uploaded_file)
    if set(expected_columns).issubset(df_input.columns):
        preds = model.predict(preprocessor.transform(df_input))
        st.write("Predictions:", preds)
    else:
        st.error("CSV должен содержать признаки: " + ", ".join(expected_columns))
```

В `expected_columns` перечислены все 30 входных признаков. Если столбцов не хватает, показывается сообщение об ошибке.

## Развёртывание и требования

Streamlit-приложение можно запускать локально или развёртывать на сервере (например, используя Docker или хостинг для Streamlit). Для корректной работы требуется:

- **Python 3.7+** (с учётом зависимостей библиотек).
- Около **500–1000 MB оперативной памяти**, поскольку набор относительно небольшой и модели (особенно XGBoost/CatBoost) помещаются в такую область. GPU не требуется для инференса (он нужен лишь для ускорения тренировки, но с такими объёмами можно обойтись CPU).
- Дискового пространства пару сотен мегабайт (модели, библиотеки).
- При промышленном развертывании: SSL, доменное имя и безопасность (см. ниже).

## Тестирование и отладка

Для тестирования обеспечьте корректность установки библиотек и версий. Частые проблемы и рекомендации:

- **Ошибки импорта:** Убедитесь, что вы активировали виртуальное окружение и установили все библиотеки (`pip install -r requirements.txt`).  
- **Проблемы загрузки моделей:** При загрузке pickle-файлов может потребоваться та же версия Python/библиотек, в которой они были созданы【46†L199-L204】. При ошибках можно пересохранить модель в новом окружении или использовать другие форматы.  
- **Streamlit пустой экран/зависание:** Проверьте, не блокирует ли файрвол порт (обычно 8501). Убедитесь, что `app.py` корректно импортирует артефакты (проверьте относительные пути).  
- **Валидация CSV:** Тестируйте на разных CSV-файлах. Убедитесь, что они закодированы в UTF-8 без BOM и имеют разделитель `,`. При любых исключениях обрабатывайте их блоками `try/except` и информируйте пользователя об ошибках.
